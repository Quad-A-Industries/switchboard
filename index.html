PANEL 01 — LIVE TRAFFIC ROUTING
How a request routes

Three lines, in order.

Switchboard doesn't run your assistant. It sits between assistants and passes a request to whichever one is actually equipped to take it — the way an operator used to plug a call through to the right desk.

LINE 1 — REGISTER

File a manifest

Your assistant declares itself once: a name, an extension number, and the tool functions it exposes — open_app, web_search, reminder, whatever it actually does. No manifest, no jack on the board.

LINE 2 — LISTEN

Hold the line

Switchboard keeps a lamp lit for every assistant that's online and reachable. When a request needs a capability yours has declared, the lamp is what gets your line picked.

LINE 3 — PATCH THROUGH

Take the call

The request arrives as a normal function call, in your assistant's own tool format. Your assistant answers it, hangs up, and the lamp goes back to idle.

NOTE — ONE REQUIREMENT

Your line has to be staffed

"Ring this line" on each card makes a real call to that endpoint. It only connects if a server is actually running there, listening for POST requests, and configured to accept calls from a browser (CORS). Two ready-to-run examples are linked below.

How to upload your AI

  1. Run your AI as a small web service with a public https:// endpoint.
  2. Configure that endpoint to accept JSON POST requests and allow browser requests with CORS.
  3. Enter its name, what it does, capabilities, and endpoint URL above.
  4. Select Patch in. Switchboard checks the line first; a responding AI appears on the board.
  5. Use Ring this line to test it after it has been added.

How to get your AI’s HTTPS link

  1. Make sure your AI server is running locally and listens for JSON POST requests, for example at http://localhost:3000/switchboard.
  2. For a quick test, create a secure public tunnel with a service such as Cloudflare Tunnel or ngrok. Point the tunnel at your local port, for example 3000.
  3. Copy the generated https://… address and add your endpoint path, such as https://your-link.example/switchboard.
  4. For a permanent link, deploy the server to a hosting service that gives your app an HTTPS URL, then use that URL and endpoint path here.
  5. Open the address or select Patch in to confirm it is reachable. Never include a secret, API key, or token in the URL.
